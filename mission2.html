Mission 2 : compréhension et fiabilisation de la base de données Kdou

1. Exploitation BDD avec une interface low-code

Cette mission avait pour objectif d’exploiter la base de données KDou à l’aide de Microsoft Access, un outil low-code permettant de manipuler des données relationnelles sans programmation complexe.

Grâce au QBE (Query By Example) et au langage SQL, nous avons conçu différentes requêtes destinées à extraire des informations précises : analyse des clients, des fournisseurs, des catégories de produits ou encore de l’historique des commandes.

L’objectif final était de faire ressortir des indicateurs pertinents pouvant aider KDou à optimiser son offre, améliorer sa gestion commerciale et comprendre les comportements d’achat.

2. Indicateurs

Clients n’ayant jamais passé de commande

Cet indicateur identifie les clients présents dans la table Clientèle mais absents de la table Commande. Cela permet à l’entreprise de comprendre quels clients sont inactifs, peut-être par manque d’information, par désintérêt ou à cause d’une offre inadéquate.

Ce résultat est essentiel pour améliorer la fidélisation, ajuster la communication commerciale et relancer des clients potentiels.

Code SQL utilisé :


SELECT Commande.CodeCli, Clientele.Societe, Clientele.MailCli, Clientele.CodeCli
FROM Clientele
LEFT JOIN Commande ON Clientele.CodeCli = Commande.CodeCli
WHERE Commande.CodeCli IS NULL;

Interprétation :

Ce résultat montre quels clients ne génèrent aucune commande malgré leur présence dans la base. Ils représentent un potentiel inexploité pour KDou :

  • Relance commerciale ciblée ;
  • Offres découvertes ;
  • Analyse de leur profil pour comprendre leurs freins d’achat.

Fournisseurs polyvalents

Cet indicateur identifie les fournisseurs capables de proposer des produits de deux catégories différentes : Viandes et Poissons & fruits de mer.

Un fournisseur polyvalent est particulièrement stratégique pour l’entreprise car il permet :

  • de réduire les coûts logistiques ;
  • de sécuriser plusieurs gammes de produits ;
  • de simplifier la gestion des stocks ;
  • et de négocier plus facilement des conditions commerciales.

Code SQL utilisé :

SELECT F.*
FROM FOURNISSEUR AS F
WHERE F.NoFour IN (
    SELECT P.NoFour
    FROM PRODUIT AS P
    WHERE P.CodeCateg = 8
)
AND F.NoFour IN (
    SELECT P.NoFour
    FROM PRODUIT AS P
    WHERE P.CodeCateg = 6
);

Interprétation :

Ces fournisseurs constituent des partenaires stratégiques. Leur polyvalence leur permet de répondre à plusieurs besoins :

  • Optimisation des commandes,
  • Réduction des délais,
  • Amélioration du service,
  • Sécurisation des approvisionnements.

3. Gestion des erreurs

Lors de l’exécution des requêtes, nous avons rencontré quelques erreurs SQL ou de noms de champs, notamment à cause de caractères spéciaux comme N° commande. Ces erreurs ont été corrigées en encadrant les noms de champs entre crochets [ ].

4. Conclusion

Cette mission nous a permis de mieux comprendre comment exploiter une base de données relationnelle à travers Access et SQL.

Les indicateurs développés montrent qu’il est possible d’obtenir rapidement des informations clés sur :

  • l’activité réelle des clients,
  • la performance des fournisseurs,
  • et la structure de l’offre produit.
